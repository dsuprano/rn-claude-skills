# Conventions

## Reglas generales
- Cada skill vive en `skills/<name>/SKILL.md`
- El `name` del frontmatter define el comando `/name`
- Toda skill debe tener descripción clara
- Toda skill debe ser reutilizable
- Toda skill debe poder aplicarse al proyecto completo o a un scope

## Convención de nombres
- `rn-arch-audit`
- `rn-cleanup`
- `rn-quality-gate`
- `rn-test-guard`

## Objetivo
Mantener un catálogo de skills reutilizables para React Native + TypeScript + MobX + React Navigation.
EOF
